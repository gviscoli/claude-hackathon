Architecture

[Incoming Claim JSON] → channels: email / web / mobile / fax
             │
[Coordinator Agent] <── lookup_policy · claim_history · fraud_check · assess_damage
             │
 ┌──────────────┼──────────────┐
[auto-claims]  [property-claims]  [medical-claims]
             │
[PreToolUse Hook] ← blocks if payout > €50k or fraud ≥ 0.7
             │
[processed_claims/{claim_id}.json] + audit.log

Inference provider:  Anthropic API  |  AWS Bedrock  (selezionato via CLAUDE_CODE_USE_BEDROCK)

Decision Logic

DecisionConditionsExample
✓ fast-track Amount ≤ €10k, fraud < 0.3, docs attached, policy active Rear-end collision, police report, first claim
? investigate Fraud 0.3–0.69, amount €10k–€50k, incomplete docs Second property claim in 2 years, €28k flood
✗ deny Inactive policy, claim type not covered, fraud ≥ 0.7 Expired policy, medical claim on auto policy
! escalate-human Payout > €50k, confidence < 0.6, fraud flag + suspicious pattern "Total loss, no docs, 3rd claim, urgent cash settlement"

Safety Rails — PreToolUse Hooks

🛡️

High-Value Block

Intercepts create_claim_record.py calls. Denies auto-approve if payout > €50,000. Injects system message to force escalate-human.

🔒

Fraud Gate

Fraud score ≥ 0.7 → automatic escalation before any write action. Deterministic — no LLM involved.

📋

Audit Log

Every Bash tool call logged asynchronously to audit.log. Non-blocking, JSONL format for easy parsing.

🚫

PII Protection

Blocks any attempt to send policies.json or customer data to external endpoints.
